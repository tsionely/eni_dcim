 219: # SIM OPERATOR Runbook
 220: 
 221: Instructions for an AI agent (Sakana, Claude Code, or any other) running on the
 222: **Windows machine where FlightSim.exe is installed**. You are the *sim
 223: operator* half of a two-agent team:
 224: 
 225: - **You (local agent)**: operate the real simulator, run flights, collect
 226:   artifacts, commit them to this repo.
 227: - **Cloud agent**: analyzes your artifacts, develops/fixes/upgrades the pilot
 228:   code in `src/`, pushes code changes. It cannot reach the simulator — your
 229:   artifacts are its only window into the real sim.
 230: 
 231: All coordination happens through git on branch `main`.
 232: 
 233: ## Ground rules
 234: 
 235: 1. **Never edit `src/`, `simtools/`, `tests/`, `docs/` or `config/`** — code
 236:    and parameter changes are the cloud agent's job. If you believe a code
 237:    change is needed, describe it in your run report instead.
 238: 2. **Only add files under `fixtures/`** (created by `scripts/collect_artifacts.py`).
 239: 3. Commit messages start with `[sim-run]`, e.g. `[sim-run] phase1 vs sim v1.0.3385`.
 240: 4. Always `git pull` before running — the cloud agent may have pushed fixes
 241:    for issues found in your previous run.
 242: 5. Recording files larger than ~50 MB: do NOT commit; upload to the Google
 243:    Drive folder `AI-GP Simulator/recordings` instead and mention the filename
 244:    in your report.
